Twój agent rozumie
Twoje pieniądze.

BR Budget ma natywne API dla agentów. Generujesz AGENT_API_KEY, podajesz agentowi — dalej działa sam: dodaje transakcje, prowadzi dwuetapowy import (preview → commit), czyta pulpit. Ty mówisz po polsku, agent rozmawia z systemem za Ciebie.

  • Bearer token na konto.Jeden token przypisany do użytkownika. Agent widzi tylko Twoje konta, kategorie i transakcje. Skill dla openClaw / Hermes drop-in.
  • Dwuetapowy import./api/agent/import/preview → wiersze, sugestie, duplikaty, pokrycie miesiąca. /api/agent/import/commit → zapis dopiero po decyzji.
  • Transakcje i pulpit.POST /api/agent/transactions dodaje wydatek z kategorią. GET /api/agent/summary zwraca miesiąc, salda i kluczowe statystyki dla coacha.
  • Wymuszenie kategorii.Agent nie commituje wydatków bez kategorii — API odpowie błędem z listą brakujących wierszy. Mechanika no-spend zostaje czysta.

· Mechanika no-spend
i.

Tylko pula zmienna.

Kategorie dzielą się na fixed (rachunki, kredyty) i variable (restauracje, rozrywka, impulsy). Mechanika no-spend liczy tylko zmienne — koszty stałe nie mieszają się z grą.

ii.

Oszczędnościomierz w zł/h.

Pozostała pula zmienna podzielona przez liczbę godzin do końca miesiąca. Nie wydajesz — licznik rośnie. Wydatek zmienny — licznik spada. Bez arbitralnych mnożników.

iii.

Codzienny rytuał „Zamknij zysk".

Rano system liczy wczorajszy wynik. Jeśli dodatni — tworzy zadanie przelewu na procent zapasu. Jeden ekran, jedno zdanie, jeden przycisk: Przelałem.

iv.

Streak za realny przelew.

Streak rośnie tylko po kliknięciu Przelałem, nie za samo niewydawanie. Pieniądze faktycznie zmieniają konto — gamifikacja bez ściemniania.
